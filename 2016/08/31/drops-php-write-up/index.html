查看源代码 

<!--
if (!isset($_GET['number'])) die('wrong request!');
$number = $_GET['number'];
$flag="xxx";

die("Length of string must more than 5 ");

echo $flag;
else

可知 最少五位可number应为4444所以采用00截断 number=4444%004

if (isset($_GET['which']))

源代码

<!--
$one = ord('1');
$nine = ord('9');
$flag = 'xxx';
if(isset($_GET['key']))
{
$number = $_GET['key'];
for ($i = 0; $i < strlen($number); $i++)
{
    $digit = ord($number{$i});
    if (($digit >= $one) && ($digit <= $nine))
        return False;

发现它是逐个过滤 不能出现1-9 但是number==’11259375’ 有点蒙蔽 弄了半天都没明白 无意间发现这个地方有相似的题 原来是php黑魔法

输入key=0xabcdef 得到flag{Yesterday_Is_History}

php-4
源代码

<!--
$flag = "xxx";
if (isset($_GET['name']) and isset($_GET['password'])) {
if ($_GET['name'] == $_GET['password'])
    print 'Your password can not be your name.';
else if (md5($_GET['name']) === md5($_GET['password']))
    die($flag);
else
    print 'Invalid password';
}
-->

这里发现第一个是==第二个是=== 就去百度了一下两者的区别

==是比较运算，它不会去检查条件式的表达式的类型
===是恒等，它会检查查表达式的值与类型是否相等

越看越熟悉 发现这一题在第三题的网站里也有相似的题目

name[] = 1和password[]= 2相比较，可以跳过第一个判断，而如果使用md5对一个数组进行加密，返回的将是NULL，NULL===NULL，这是成立的，所以构造两个数组，成功拿到flag

然而我并没有看明白= = 尝试在界面填写 一直出错 然后观察URL发现在输入框里写无法构成数组 然后直接在url里写name[]=1&password[]=2得到flag{Array_IS_Fun} = = 得到结论：当md5一个数组的时候，得到的结果是一个空值。那么2个数组的md5是相等
